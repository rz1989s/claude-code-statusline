## рҹ”Қ What Each Line Shows

Understand every element of your enhanced statusline with this detailed breakdown:

### рҹ“Ғ **Line 1: Repository & Environment Info**

```
~/local-dev (main) вң… в”Ӯ Commits:0 в”Ӯ ver2.1.45 в”Ӯ SUB:вҖ” в”Ӯ рҹ•җ 08:22
```

- **рҹ“Ӯ Directory**: Current working directory with elegant `~` notation
- **рҹҢҝ Git Branch**: Active branch name with visual status indicators
- **вң… Status**: Clean (вң…) or dirty (вҡ пёҸ) repository state
- **рҹ“қ Commits**: Today's commit count for productivity tracking
- **рҹ”ў Version**: Claude Code version (intelligently cached for performance)
- **рҹ“Ұ Submodules**: Git submodule count (shows `вҖ”` when none)
- **рҹ•җ Time**: Current system time for session awareness

### рҹ’° **Line 2: Cost Tracking & Model Info**

```
рҹҺө Sonnet 4 в”Ӯ REPO $3.87 в”Ӯ 30DAY $108.81 в”Ӯ 7DAY $66.48 в”Ӯ DAY $9.35 в”Ӯ рҹ”Ҙ LIVE $6.74
```

- **рҹҺө Model**: Current Claude model with distinctive emoji indicator
- **рҹ“Ҡ REPO**: Total cost for current repository session
- **рҹ“… 30DAY**: Monthly spending total across all sessions
- **рҹ“Ҳ 7DAY**: Weekly spending for budget tracking
- **рҹҢ… DAY**: Today's accumulated costs
- **рҹ”Ҙ LIVE**: Active billing block cost (when block is active)

*Powered by [ccusage](https://ccusage.com) for accurate cost monitoring*

### рҹ”Ң **Line 3: MCP Server Health**

```
MCP (3/4): upstash-context-7-mcp, supabase-mcp-server, firecrawl-mcp, sqlscan-mcp
```

- **рҹ“Ў Status Count**: Connected servers vs total configured servers
- **рҹ“Ӣ Server List**: Individual MCP server names
- **рҹҹў Connection Status**: Color-coded health indicators
  - рҹҹў **Connected**: Server is healthy and responding
  - рҹ”ҙ **Disconnected**: Server is down or unreachable
- **вҡЎ Real-time**: Updates automatically as servers come online/offline

### вҸ° **Line 4: Block Reset Timer** *(Context-Aware Display)*

```
RESET at 11.00 (2h 37m left)           # Normal countdown
RESET at 06.00 (waiting API response...)  # API calculating projection
(Hidden when no active block)              # No active billing block
```

- **рҹ•’ Reset Time**: When current 5-hour conversation block expires
- **вҸі Smart States**: Three intelligent display modes:
  - **рҹ“Ҡ Active Countdown**: `(4h 15m left)` when projection data available
  - **вҸі API Processing**: `(waiting API response...)` during calculation delays  
  - **рҹ”Ү Hidden Display**: Automatically hidden when no active block
- **рҹҺҜ Enhanced Detection**: Validates both block status and projection data
- **рҹ“… Context Aware**: Handles API timing issues gracefully
